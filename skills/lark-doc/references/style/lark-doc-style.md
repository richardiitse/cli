# 文档样式指南

创建或编辑文档内容时，必须遵循本指南，使用结构化 block 提升可读性和视觉层次。

## 一、核心原则

1. **结构优于文字**：能用结构化 block 表达的信息，不用纯文本段落
2. **Front-load 结论**：每篇文档以 `<callout>` 开头概括核心结论；每个章节首段点明本节要旨
3. **视觉节奏**：连续纯文本不超过 3 段，用富 block 打断单调感；不同主题章节间用 `<hr/>` 分隔
4. **最少惊讶**：同类信息使用同类元素，全篇风格统一


## 二、元素选择指南

优先使用结构化 block 代替纯文本段落。涉及图表需求时，推荐多使用 **lark-whiteboard** skill（飞书画板）——支持无限画布、自由布局、连线、分组，表达力远强于文档内嵌元素；简单图也可用 `<whiteboard type="mermaid/plantuml">` 快速内嵌。

### 文本与结构类

| 场景 | 推荐方案 | 避免 |
|-|-|-|
| 核心结论 / 摘要 / 注意事项 | `<callout>` + emoji + 背景色 | 纯 `<p>` 加粗 |
| 方案对比 / 优劣势 / Before vs After | `<grid>` 2 列分栏 | 用文字描述"左边…右边…" |
| 3+ 属性的结构化数据 / 指标表 | `<table>` + 表头背景色 | 用列表罗列 |
| 任务清单 / 检查项 | `<checkbox>` | `- [ ]` 文本或普通列表 |
| 章节分隔 / 视觉节奏 | `<hr/>` | 连续空行 |
| 代码片段 | `<pre lang="x" caption="说明">` | 行内 `<code>` 包大段代码 |
| 引用 / 公式 | `<blockquote>` / `<latex>` | 纯文字写公式 |
| 操作入口 / 跳转链接 | `<button>` / `<a type="url-preview">` | 裸 URL |

### 图表类

简单图优先内嵌 `<whiteboard type="mermaid/plantuml">`，复杂图使用 **lark-whiteboard** skill 独立画板。

| 场景 | 推荐方案 | 避免 |
|-|-|-|
| 时序图 / 简单流程图 / 状态机 / 类图 | `<whiteboard type="mermaid/plantuml">` | 纯文字描述流程 |
| 甘特图 / 排期 / 里程碑 | `<whiteboard type="mermaid/plantuml">` | 用表格模拟时间线 |
| 架构图 / 系统拓扑 / 部署图（需详细标注） | **lark-whiteboard** skill | 用文字或代码块表达复杂关系 |
| 组织架构图 / 汇报关系 | **lark-whiteboard** skill | 用缩进列表模拟层级 |
| 数据图 / 趋势图 / 饼图 / 柱状图 | **lark-whiteboard** skill | 用表格模拟图表 |
| 漏斗图 / 鱼骨图 / 金字塔 / 飞轮图 | **lark-whiteboard** skill | 用文字描述结构 |
| 思维导图 / 脑图 / 知识树 | **lark-whiteboard** skill | 多级列表模拟树形 |

### 快速判断流程

```
需要表达的信息 →
  ├─ 是对比关系？ → <grid> 分栏
  ├─ 有 3+ 字段的结构化数据？ → <table>
  ├─ 是流程/关系/拓扑？
  │    ├─ 能用 mermaid/plantuml 5 行内表达？ → <whiteboard> 内嵌
  │    └─ 更复杂？ → lark-whiteboard skill
  ├─ 是重要结论/警告/提示？ → <callout>
  ├─ 是待办事项？ → <checkbox>
  └─ 以上都不是 → <p> 段落（但注意不超过连续 3 段）
```



## 三、颜色语义

善用颜色系统区分信息类型，全篇保持语义一致：

| 语义 | emoji 前缀 | callout 背景色 | 文字 / 边框色 | 典型场景 |
|-|-|-|-|-|
| 信息、说明 | ℹ️ "说明：" | `light-blue` | `blue` | 背景知识、补充说明 |
| 成功、推荐 | ✅ "推荐：" | `light-green` | `green` | 最佳实践、已完成项、正向指标 |
| 警告、风险 | ⚠️ "警告：" | `light-red` | `red` | 已知问题、阻塞项、负向指标 |
| 错误、反模式 | ❌ "错误：" | `light-red` | `red` | 常见错误、反模式示例 |
| 注意、待确认 | ❗"注意：" | `light-yellow` | `yellow` | 待讨论项、未确认假设、易遗漏要点 |
| 中性、辅助 | — | `light-gray` | — | 表头背景、次要信息 |

补充规则

- 表头统一使用 `background-color="light-gray"`
- 关键指标用 `<span text-color="green/red">` 突出正负变化，**但必须同时用 ↑↓ 或 +/- 标注方向**，确保色觉障碍用户也能识别正负含义
- 同一文档中，同一语义必须使用同一颜色，不可混用


## 四、排版节奏

- **标题层级** ≤ 4 层（h1 → h4），超过说明结构需要拆分
- **段落长度**：单段不超过 5 行，超过时拆分或提取结构化元素
- **列表深度**：嵌套 ≤ 2 层，超过时考虑用 table 或拆分章节
- **Grid 列数**：最多 3 列，推荐 2 列对比
- **空行**：不同 block 类型之间用空行分隔
- **文档开头**：用 `<callout>` front-load 最重要的结论或摘要
- **章节结尾**：长章节末尾可用 `<callout>` 小结要点


## 五、丰富度自检

生成文档内容后，用以下指标自检丰富度，**未达标时必须主动优化**：
| 指标 | 达标标准 | 计算方式 |
|-|-|-|
| **富 block 密度** | ≥ 40% | 非纯文本 block 数 ÷ 总 block 数（纯文本 = `<p>` / `<ul>` / `<ol>`） |
| **元素多样性** | ≥ 3 种 | 文档中使用的不同 block 类型数（callout / grid / table / whiteboard / checkbox / hr / pre 等各计 1 种） |
| **连续纯文本上限** | ≤ 3 段 | 连续 `<p>` 段落不超过 3 段，超过时用 callout 提炼、table 结构化或 grid 对比 |
| **章节丰富度** | 每章 ≥ 1 | 每个 h1/h2 章节至少包含 1 个非纯文本 block |
| **开头 callout** | 必须 | 文档开头用 `<callout>` front-load 核心结论或摘要 |
| **视觉节奏** | 有 `<hr/>` | 不同主题章节间用 `<hr/>` 分隔 |


## 六、常见反模式

以下是审查文档时需要识别并修正的典型问题：

| 反模式 | 问题 | 修正方式 |
|-|-|-|
| "文字墙" | 连续 5+ 段纯文本，无视觉锚点 | 提取关键信息为 callout / table / grid |
| "列表地狱" | 用 3 层嵌套列表表达复杂结构 | 改用 table 或拆分为多个平级章节 |
| "伪表格" | 用制表符/空格对齐模拟表格 | 使用 `<table>` |
| "裸链接" | 直接贴 URL 无上下文 | 使用 `<a type="url-preview">` 或 `<button>` |
| "彩虹文档" | 颜色无语义，随意使用 | 严格按第三节颜色语义规范 |
| "标题缺失" | 长文档无章节结构 | 按逻辑分块，添加 h2/h3 标题 |
| "信息过载" | 单个 table 超过 10 行无分组 | 拆分为多表或增加分组标题行 |
